初めてのHello World (3) - コンパイルと実行

　今回は、前回書いたコードをコンパイルして、実行してみます。このソースコードを「hello.c」という名前で保存してください。 

<h3>include <stdio.h></h3>

// エントリポイント
int main () {
  
  // Hello World!を出力
  printf("Hello World!");
  
  // 正常に終了したことを知らせる
  return 0;
}

ソースファイルのコンパイル

　Perlでは、作成したスクリプトは「perl」コマンドで直接実行することができました。C言語では、コンパイルという作業が必要になります。

　コンパイルとは、ソースコードを、機械語で書かれたオブジェクトファイルと呼ばれるファイルに変換する作業のことです。最終的に、オブジェクトファイルは、実行ファイルに変更されますが、一段階目の作業として、コンパイルを行う必要があります。 

　　　　　　(コンパイル)
ソースファイル　→　オブジェクトファイル

　コンパイルを行うには「gcc」というコマンドの「-c」オプションを使用します。 

gcc -c hello.c

　すると「hello.o」というオブジェクトファイルが作成されます。 

hello.o

実行ファイルの作成

　今はひとつのソースコードから、ひとつのオブジェクトファイルを作成しましたが、規模が大きくなって、複数のソースコードから、複数のオブジェクトファイルが作成されたと想像してみてください。 

code1.o
code2.o

　実行ファイルはこれらのオブジェクトファイルを結びつけることによって作成されます。この作業をリンクと呼びます。 

code1.o　+　code2.o →　code(実行ファイル)

　リンクを行って実行ファイルを作成するに「gcc」の「-o」コマンドを使用します。 

gcc -o 生成される実行ファイル名 オブジェクトファイル1 オブジェクトファイル2

　今回はオブジェクトファイルがひとつですので、次のようにします。 

gcc -o hello hello.o

　次のように実行ファイルが作成されます。Linuxにおいては「hello」という名前、Windowsにおいては「hello.exe」という名前になります。 

<h3> Linux</h3>
hello

<h3> Windows</h3>
hello.exe

実行ファイルを実行する

　では実行ファイルを実行してみましょう。Linuxでは、カレントディレクトリにある実行ファイルは、「./hello」のように「./」を前につけて実行する必要があります。Windowsでは「hello」あるいは「hello.exe」で実行できます。 

<h3> Linux</h3>
./hello

<h3> Windows</h3>
hello
hello.exe

　これで、コンパイルして、プログラムを実行できるようになりました。
