初めてのHello World (1) - ライブラリの読み込み

　ではさっそく「Hello World」をC言語で記述してみましょう。C言語のソースコードの拡張子は「.c」です。「hello.c」というファイルに次のコードを書いてみてください。

```c:

#include

// エントリポイント

int main () {

// Hello World!を出力 printf("Hello World!\n"); // 正常に終了したことを知らせる return 0;

}

```

##ライブラリの読み込み

　まず最初にライブラリの読み込みについて見てみましょう。

```c:

#include

```

　Perlではuseを使ってモジュールを読み込みました。C言語では、ライブラリの読み込みはincludeディレクティブを使って行います。ディレクティブというのは、プリプロセッサへの命令のことです。プリプロセッサ命令は「#」で始まります。

　プリプロセッサとは、ソースコードをコンパイルする前の前処理を行うものです。C言語では、ライブラリの読み込みというのは、他のソースコードを取り込むことに等しいです。

　つまり「stdio.h」というソースコードを現在のソースコードに取り込むということを意味しています。コンパイルまでの流れを見ると次のようになります。

```

ソースコード　→　前処理されたソースコード　→　コンパイルされたソースコード

　　　　(プリプロセッサ)　　　　　　 (コンパイラ)

```

###のように「<>」で囲まれているのはどんな意味があるのか

　どうして「stdio.h」は「<>」で囲まれているのだろうと不思議に思います。C言語ではルールとして、標準ライブラリは「<>」で囲むというルールがあります。

　「stdio.h」は、標準ライブラリなので「<>」で囲んでいます。自作のライブラリの場合は次のように、ダブルクォーテーションで囲みます。

```c:

#include "mylib.h"

```

###stdio.hの「.h」はどんな意味があるのか

　C言語では、ライブラリは、ソースファイルとヘッダファイルに分けて書くことが一般的です。ソースファイルは「stdio.c」になります。ヘッダファイルは「stdio.h」になります。

　簡潔に説明すると、ヘッダファイルには、関数の宣言を書きます。ソースファイルには、関数の実装を書きます。ライブラリを利用するためには、ヘッダファイルを読み込むだけで大丈夫です。最終的には、コンパイルされたソースファイルが結び合わされることになりますが、ソースコードを書く上では、ヘッダファイルを読み込むだけで、大丈夫です。

###stdio.hはどこに存在するのか

　Linux環境であれば「/usr/include」にあります。Windowsであれば「C:\Perl\site\lib\auto\MinGW\include」にあります。

###stdioは何をするためのライブラリか

　stdioは、標準入出力を扱うためのライブラリです。printf関数を利用するためには「stdio」を読み込む必要があります。
